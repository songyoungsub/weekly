CSS의 Cascading에 대해

Cascading은 폭포, 위에서 아래로 쏟아지는이라는 뜻으로 CSS에서 가장 중요한 스타일 적용 규칙이다.
CSS를 사용하다보면 같은 요소에 대해 여러개의 스타일이 중복될 수 있다.
예로
                <div style='background-color: #f00'>

                    <p style='background-color: #0f0'>캐스케이딩 설명 중</p>

                </div>
p는 div의 하위 요소로 div에 적용된 스타일은 하위 요소들에 적용이 됩니다. 하지만 하위 요소인 p에도 스타일이 따로 지정되어 있습니다.
이런 경우가 빈번하게 일어나자 CSS는 스타일을 적용할 규칙을 세우기 위해 크게 두 개의 캐스케이딩 규칙을 채택하기로 했다.

스타일 우선순위

첫번째 규칙으로 스타일 우선순위는 또 중요도, 명시도, 코드 순서로 나뉜다.

중요도

스타일이 선언된 중요도 따라서 우선순위를 매기는 것으로 우선순위는 다음과 같다.
사용자 스타일 시트 > 작성자 스타일 시트 > 브라우저 스타일 시트

사용자 스타일은 최종 사용자가 브라우저 상에서 설정한 옵션(윈도우 테마 설정, 고대비 설정 등)을 의미하고
작성자 스타일은 웹 문서를 개발한 사람이 설정한 스타일을,
브라우저 스타일은 별다른 스타일 선언 없이 웹 문서를 작성했을 때 크롬이나 사파리 등이 디폴트로 가지고 있는 스타일을 의미한다.

위 순서대로 중용도를 가지지만 의도적으로 중요도를 올릴 수 있다.
그 방법은 '!important' 키워드를 사용하는 것이다. 이 키워드를 사용하면 중요도가 최우선으로 끌어올려진다.

선택자 {
속성: 속성값 !important
}

이런식으로 사용하면 된다.

명시도

우선순위를 결정하는 두 번째 요소는 명사도로 선택자 즉, 셀렉터가 가리키는 것이 명확할수록 우선순위를 높게 주는 것을 의미한다.
명시도에 따른 우선 순위는 다음과 같다.

인라인 > id > class > 태그

인라인은 하나의 태그(HTML 파일 내에서 작성) 내에서만 적용되므로 높은 우선순위를 갖습니다.
id는 문서 내에서 한 번만 등장하므로 비교적 좁은 범위를 가지게됩니다.
class는 여러번 쓰일 수 있기에 id보다는 넒은 범위를 갖습니다.
태그는 문서 내 모든 태그를 가리키므로 범위가 제일 넒습니다.

코드 순서

우선순위의 마지막 요소는 코드 순서이다.
코드 순서는 코드에서 가장 마지막에 등장한 속성을 최우선으로 적용한다는 것으로 늦게 작성한 속성일수록 우선 순위가 높다.

정리하면 우선순위는 인라인 스타일 > !important를 사용한 스타일 > 사용자 스타일 시트 > 작성자 스타일 시트 > 브라우저 스타일 시트이며
여기서 인라인 스타일을 제외하면 id > class > 태그 순으로 우선순위가 높고 명사도가 같으면 늦게 작성된 스타일이 우선 순위가 더 높다.

스타일 상속

두번째 규칙으로는 스타일 상속이다.

스타일 상속은 태그들이 어떻게 포함되었는지에 따라서 스타일을 적용할 지 결정하는 원칙이다.
                <div>
                <div>
                </div>
                </div>
위와 코드에서 바깥쪽의 태그를 부모 요소, 내부의 태그를 자식 요소라 한다.
스타일 상속은 부모 요소의 스타일을 자식 요소가 따라가는 것을 의미한다.

시맨틱 태그를 사용하면 좋은 점

시멘틱 태그란?

시멘틱 태그의 시멘틱(semantic)은 '의미의, 의미론적인'이라는 뜻을 가진 단어로 시멘틱 태그는 태그에 의미를 부여한다? 이정도로 이해하면 좋다.
우리가 자주 사용하는 div 태그로만 코드를 작성 시 나중에는 각각의 태그들이 뭐를 의미하는지 알기 힘들다.
그런것을 방지하기 위해 시멘틱 태그를 사용한다.

시맨틱 태그에는 종류가 엄청 많은데, 이는 두가지로 분리할 수 있다.
어떤 내용을 감싸는 'Box', 내용에 해당되는 'Item'이다 

                <div>
                <button>버튼</button>
                </div>
위 코드의 div 태그가 Box 역할이며, button 태그가 item 역할이다.
Box 는 쓰이는 영역을 구분 짓는 것에 쓰이는 시맨틱 태그이며, ITEM 어떤 내용에 해당되는 것들의 역할을 부여한 시맨틱 태그이다.

시멘틱 태그를 쓰는 이유

탐색이 쉽다.

div태그를 계속해서 남발하는 것은 어떤 부분에서 error가 발생시에 찾기가 매우 힘들어진다. 각각을 용도에 맞게 분리시킨 코드는 error가 터졌을때, 탐색이 쉬워진다.

검색엔진 최적화

우리가 구글, 네이버 등등에 검색을 할때, 최상위에 뜨는 것들이 있다. 이것이 검색엔진 최적화가 잘되있는 사이트들이다.
검색엔진은 태그를 기반으로 페이지 내 검색 키워드의 우선순위를 판단하는데. 예를 들어 제목을 의미하는 h태그가 div 태그보다 우선순위가 높아, 검색시에 더 잘 검색이 된다.

시각장애인분들한테 도움이 된다. (스크린 리더)

스크린 리더는 시각장애인이 눈으로 보는 대신, 소리로 들려주면서 화면에 담긴 정보를 전해준다.

만약 button 태그가 아니라, div 태그로 버튼 기능을 만든다고 생각하자. div 태그로도 충분히 버튼을 만들 수 있지만 이는 스크린 리더에서 버튼으로 인식하지 않는다.
시각장애인 분들께 서비스를 이용하는데, 매우 치명적일 수 있다.

다른 개발자들이 우리의 코드를 재활용하기 편하다.

코드를 혼자 개발하는 것이 아니라, 모두 협업해서 하는 경우가 많은데 이때 어떤 개발자가 코드를 보고 이게 어떤 코드인지 알 수 있는것이 중요하다.
이를 위해서라도 용도에 맞는 시맨틱 태그를 골라, 사용하는 것이 좋다

정리

시맨틱 태그는 말그대로 의미로 전달해주는 태그로, 태그가 어떤 태그다라는 역할을 지정해준다.
코드라는 것은 본인만 보는 것이 아니기 때문에, 다른 사람들의 가독성을 위해서라도 용도에 맞는 태그 사용은 중요하다.
